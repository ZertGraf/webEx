# АИС «Электронная библиотека»

[![Deploy](https://github.com/ZertGraf/webEx/actions/workflows/deploy.yml/badge.svg)](https://github.com/ZertGraf/webEx/actions/workflows/deploy.yml)

Автоматизированная информационная система для ведения реестра книг: каталог с
обложками и жанрами, рецензии читателей, разграничение прав по ролям и журнал
посещений со статистикой (**вариант 4**).

Учебный проект. Группа 241-327 — Мясников Дмитрий Сергеевич.

Полный текст задания: [`docs/TASK.md`](docs/TASK.md).

## Возможности

- **Каталог книг** на главной с пагинацией (10 на страницу), сортировкой по году
  (сначала новые), средней оценкой и числом рецензий по каждой книге.
- **Карточка книги** с обложкой, описанием в Markdown, списком рецензий и средней
  оценкой.
- **CRUD книг** с загрузкой обложки и мультивыбором жанров; форма создания и
  редактирования собрана из общего Jinja-макроса.
- **Рецензии** — один отзыв на книгу от пользователя, с оценкой 0–5 и текстом.
- **Роли и доступ**: администратор, модератор, пользователь. Неаутентифицированным
  доступны только главная, вход и просмотр книги; на остальное — редирект на
  страницу входа или на главную с flash-сообщением.
- **Журнал посещений (вариант 4)**: каждый просмотр карточки логируется (не более
  10 раз на посетителя/книгу в сутки). На главной выводятся «популярные за 3
  месяца» и «недавно просмотренные». В разделе «Статистика» (только админ) —
  журнал и просмотры по книгам с фильтром по датам и выгрузкой в CSV.

## Технологии

| Слой | Используется |
|------|--------------|
| Бэкенд | Python 3.12, Flask 3, Flask-SQLAlchemy, Flask-Login |
| БД | MySQL (`mysql-connector-python`); URL настраивается через `DATABASE_URL` |
| Контент | `markdown` + `bleach` (санитайзинг пользовательского ввода) |
| Фронтенд | Server-side Jinja2, Bootstrap 5.3, EasyMDE (Markdown-редактор) — через CDN |
| Прод-сервер | Gunicorn (в контейнере); как альтернатива — NGINX Unit |

## Модель данных

`roles` → `users` (роль пользователя) · `books` ↔ `genres` (многие-ко-многим через
`book_genres`) · `covers` (1:1 с книгой, дедупликация по MD5) · `reviews` (книга +
пользователь) · `visit_logs` (журнал просмотров, вариант 4). Внешние ключи на книгу
объявлены с `ON DELETE CASCADE`, поэтому при удалении книги уходят и обложка, и
рецензии, и связи с жанрами; файл обложки удаляется из ФС, если на него больше не
ссылается ни одна запись.

## Роли и тестовые учётные записи

`flask seed` создаёт роли, восемь жанров и трёх пользователей:

| Логин | Пароль | Роль | Права |
|-------|--------|------|-------|
| `admin` | `Admin123!` | администратор | полный доступ, создание/удаление книг, статистика |
| `moderator` | `Moder123!` | модератор | редактирование книг |
| `user` | `User123!` | пользователь | написание рецензий |

## Быстрый старт (локально)

Нужен Python 3.12+ и доступная база MySQL (или другая, совместимая с SQLAlchemy —
тогда поправьте `DATABASE_URL`).

```bash
python -m venv .venv
# Windows: .venv\Scripts\activate
source .venv/bin/activate

pip install -r requirements.txt

cp .env.example .env          # отредактируйте SECRET_KEY и DATABASE_URL

flask --app wsgi init-db      # создать таблицы
flask --app wsgi seed         # роли, жанры, тестовые пользователи

python wsgi.py                # dev-сервер на http://127.0.0.1:5000
```

## Быстрый старт (Docker)

Образ при старте дожидается БД, прогоняет `init-db` и `seed`, затем поднимает
Gunicorn на порту `8000`.

```bash
docker build -t webex-app .
docker run --rm -p 8000:8000 --env-file .env webex-app
```

`docker-compose.yml` рассчитан на прод-окружение с внешними сетями
(`caddy_default` — обратный прокси, `shared_mysql` — общая БД) и томом
`webex_media` для обложек. Чтобы загруженные обложки попадали в том, в `.env`
должно быть указано `UPLOAD_FOLDER=/data/covers`.

## Конфигурация

Все настройки читаются из окружения (через `.env`, см. `.env.example`):

| Переменная | Назначение | По умолчанию |
|------------|------------|--------------|
| `SECRET_KEY` | ключ подписи сессий Flask | `dev-secret-key-change-me` |
| `DATABASE_URL` | строка подключения SQLAlchemy | `mysql+mysqlconnector://library:library@localhost:3306/library` |
| `UPLOAD_FOLDER` | каталог для файлов обложек | `media/covers` (в проде — `/data/covers`) |

Загружаемые обложки ограничены 10 МБ (`MAX_CONTENT_LENGTH`).

## CLI-команды

| Команда | Действие |
|---------|----------|
| `flask --app wsgi init-db` | `db.create_all()` — создать таблицы |
| `flask --app wsgi seed` | вставить роли, жанры и тестовых пользователей (идемпотентно) |
| `flask --app wsgi seed-books` | наполнить каталог демо-книгами с обложками и рецензиями (идемпотентно) |

Контейнер прогоняет все три команды при старте, поэтому свежий деплой
поднимается уже с наполненным каталогом.

## Деплой

При пуше в `main`/`master` запускается workflow
[`.github/workflows/deploy.yml`](.github/workflows/deploy.yml): он подключается к
серверу по SSH, синхронизирует код через `rsync` (исключая `.git`, `.github`,
`.venv`, `.env`) и выполняет `docker compose up -d --build`.

Что должно быть готово заранее:

- **Секреты репозитория**: `DEPLOY_HOST`, `DEPLOY_USER`, `DEPLOY_SSH_KEY`
  (приватный ключ, чей публичный добавлен в `authorized_keys` на сервере).
- **На сервере**: установленный `docker compose`, созданные внешние сети
  `caddy_default` и `shared_mysql`, а также файл `~/webex/.env` (он намеренно не
  синхронизируется и должен лежать на сервере вручную).

Как альтернатива Docker в репозитории есть пример конфигурации NGINX Unit —
[`deploy/unit.config.json`](deploy/unit.config.json).

## Безопасность

- Пароли хранятся хешами (`werkzeug.security`).
- Markdown-описания и тексты рецензий проходят через `bleach` с белым списком
  тегов перед сохранением.
- Запись в БД и сохранение файла обложки разнесены: файл пишется только после
  успешного коммита, ошибки откатываются (`db.session.rollback()`).
- Обложки дедуплицируются по MD5 — одинаковые файлы не дублируются на диске.
